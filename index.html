Historia de las Bases de Datos

Las bases de datos surgieron para mejorar el almacenamiento y manejo de la información. En los años 60 se utilizaban archivos planos, lo que generaba muchos problemas como duplicación de datos y dificultad para acceder a la información.

En los años 70 aparecieron los modelos jerárquico y en red, que mejoraron la organización, pero eran complejos. Posteriormente, Edgar F. Codd propuso el modelo relacional, el cual revolucionó el manejo de datos al organizarlos en tablas y permitir el uso del lenguaje SQL.

Con el crecimiento de internet y el Big Data, surgieron las bases de datos NoSQL, diseñadas para manejar grandes volúmenes de información de manera flexible y escalable.

Modelo Relacional

El modelo relacional organiza los datos en tablas compuestas por filas y columnas. Cada tabla representa una entidad y se utilizan claves primarias y foráneas para relacionar la información entre tablas.

Este modelo garantiza integridad, seguridad y consistencia de los datos, y utiliza el lenguaje SQL para realizar consultas. Es ampliamente usado en sistemas académicos, empresariales y financieros.

Modelo NoSQL

El modelo NoSQL no utiliza tablas tradicionales, sino estructuras como documentos, clave-valor, columnas o grafos. Está diseñado para sistemas que requieren alta disponibilidad y escalabilidad.

Aunque es muy rápido y flexible, no siempre garantiza la integridad de los datos como el modelo relacional.
